    Cómo funciona SF Activity Tracker

    1 · Para qué sirve

    Lee tu Google Calendar de un rango de fechas, identifica con qué oportunidades de org62 se relaciona cada evento, lo clasifica por SE Task Type / CF / CR, y crea los Events + Deal Contributions con un click — sin abrir Activity Editor.

    2 · Setup inicial (una vez)

    1. Conecta Google Calendar en Settings → "Connect with Google". Te lleva al flujo OAuth en el browser. Sin terminal, sin gcloud.
    2. Selecciona qué calendarios leer. Por defecto todos; tildando solo los que importan (tu primary + los de equipo) reduces ruido.
    3. Aliases y catch-all (opcional): los va aprendiendo solo cuando corriges matchings.

    3 · Día a día

    1. Selecciona un rango (Hoy / Ayer / Esta semana / Semana pasada / Este mes o fechas custom).
    2. Pulsa ▶ Analyze. Tarda ~5-30s con Google API directo. Si tildas Force refresh ignora el cache.
    3. Revisa el calendario. Cada evento toma color según su estado (ver leyenda).
    4. Edita el draft plan de la tabla: cambia Related To, SE Task Type, CF/CR. Tilda con check los que quieres crear.
    5. Pulsa "Create N in org62". Stream de progreso, los logueados se marcan en gris automáticamente.

    4 · Leyenda de colores del calendario

    • Identified: el clasificador lo asignó a una oportunidad → se va a crear si lo tildas.
    • Already logged: ya existe un Event en org62 con ese subject + start. No se duplica.
    • Flagged: el clasificador no encontró match claro. Revisa manualmente el Related To.
    • Excluded / skip: descartado (título excluido, RSVP=declined, OOO, etc.). No se loguea.

    La franja de color a la izquierda de cada evento es el color original que tiene en Google Calendar.

    5 · Filtros y atajos

    • Click en un evento del calendario → resalta su fila en la tabla. Click otra vez → quita el resaltado.
    • Click en una fila → navega el calendario al día y resalta el evento.
    • Filter panel: caja de texto, pills de status y de color de Google Calendar.
    • Show already logged: por defecto los logueados se ocultan para vista limpia. Tilda para verlos.

    6 · Aprendizaje progresivo

    Cuando corriges un Related To en la tabla, te ofrece guardarlo como alias para que el próximo evento con ese mismo keyword se clasifique solo. Lo mismo con las correcciones de SE Task Type.

    7 · Acceso rápido desde el escritorio

    Instala los dos atajos del escritorio:

    cd ~/Documents/Salesforce/tools/sf-activity-tracker
./bin/install-desktop-shortcut.sh

    Aparecen dos iconos en tu Desktop:

    • SF Activity Tracker — comprueba si el server corre, lo arranca si hace falta, abre el browser en http://127.0.0.1:7825.
    • Restart SF Activity Tracker — fuerza un bounce del server. Útil tras cambios al código del backend o si se queda colgado.

    Pasa --symlink si prefieres que los iconos sigan actualizándose con git pull. Si no, re-corre el script tras cada pull para refrescar las copias.

    8 · Auto-start del server (launchd)

    El server corre en background como LaunchAgent: arranca al hacer login en macOS, se reinicia solo si crashea, y los logs van a ~/Library/Logs/sf-activity-tracker.log.

    Comandos en tu terminal (desde la carpeta del proyecto ~/Documents/Salesforce/tools/sf-activity-tracker):

    CuándoComando
    Tras tocar código del backend (server/)./bin/launchd-install.sh restart
    Ver si está corriendo./bin/launchd-install.sh status
    Ver logs en vivotail -f ~/Library/Logs/sf-activity-tracker.log
    Tras pull con cambios en el plist./bin/launchd-install.sh install
    Desactivar (quitar el auto-start)./bin/launchd-install.sh uninstall

    Tras cambios en frontend (public/) basta con Cmd+R en el browser, no hace falta restart.

    Si cierras el portátil totalmente el server se para hasta el próximo login. launchd corre en sesión de usuario.

    9 · Troubleshooting

    • Timeout en classify: el clasificador se trocea en lotes de 10 eventos en paralelo. Si sigue fallando, reduce el rango.
    • Backend en 🐢 Claude: significa que no tienes Google API conectado. Settings → Connect with Google.
    • Eventos faltan en el calendar: revisa Settings → "Calendarios a leer" — quizá tienes deseleccionados algunos.
    • App no responde en localhost:7825 tras reboot: ./bin/launchd-install.sh status. Si dice "Not running", install. Mira logs si crashea repetidamente.
